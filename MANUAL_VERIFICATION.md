# 手动复测指南（请按顺序）

> 目的：验证最近的渲染修复是否生效，并定位仍需优化的点位。
> 如果你遇到异常，请附上：文件路径 + 段落/行号 + 期望/实际截图。

## 1) 长文件截断（高优先级）
- 打开一份 > 200 行的 Markdown（或 `docs/tests/performance-test-500-lines.md`）。
- 期望：滚动到底部时，所有行均可见、无中途“消失”。
- 若仍截断：记录“最后可见行号”与“文件总行数”。

## 2) 文件切换内容错乱（高优先级）
- 连续打开 3 个不同文件（A/B/C），快速切换 A→B→C→B。
- 期望：每次显示对应文件内容，无串档/残影。
- 若出现错乱：记录切换顺序与错乱发生的文件名。

## 3) 表格渲染与编辑
- 覆盖以下场景：
  - 表格在引用/列表内
  - 表格内混排 `**` / `==` / `$...$` / `` `code` ``
  - 表格行包含转义管道 `\|`
- 期望：表格结构稳定，单元格内容正确解析。

## 4) 代码块围栏识别
- 测试 ``` / ~~~、带空格语言（如 ``` c++ ）。
- 测试引用/列表中的围栏代码块。
- 期望：代码块整体渲染，语言识别正确。

## 5) 水平线（HR）宽度
- 使用 `---` 与 `- - -`、`* * *`。
- 期望：横线贯穿文本区域，非“短线”。

## 6) 公式渲染（特殊位置）
- 公式位于列表项、引用、表格单元格内。
- 期望：全部渲染为 KaTeX，无空白或 “undefined”。
